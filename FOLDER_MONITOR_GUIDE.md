# 文件夹监听功能说明

## 功能概述

新增的文件夹监听功能可以自动监测指定文件夹中的新图片文件，并自动进行缺陷检测和结果显示。

### 监听模式

系统支持两种监听模式，自动选择可用的模式：

#### 1. **实时监听（推荐）**
- 使用 `watchdog` 库实现文件系统事件监听
- 文件变化时立即响应，无需等待扫描周期
- 跨平台支持（Windows/Linux/macOS）
- 低资源消耗

#### 2. **扫描监听（降级方案）**
- 定期扫描文件夹（每 2 秒）
- 作为 watchdog 不可用时的备份方案
- 兼容性好，但延迟较高

## 使用流程

### 1. 安装依赖（首次使用）

```bash
pip install -r requirements.txt
```

确保安装了 `watchdog` 库：

```bash
pip install watchdog>=3.0.0
```

### 2. 选择文件夹
点击左侧侧边栏的 **「📁 文件夹选择」** 按钮，选择要监听的文件夹。

### 3. 选择检测插件
在右侧面板的 **「选择检测插件」** 下拉框中，选择一个检测插件。

### 4. 启动监听
点击左侧侧边栏的 **「▶️ 启动监听」** 按钮。

监听状态显示：
- **「实时监听中」** - 使用 watchdog 实时监听（推荐）
- **「扫描监听中」** - 使用定期扫描（备份方案）

### 5. 自动检测
- 监听启动后，系统会监听文件系统事件
- 当检测到新的图片文件时，**立即**进行检测
- 检测结果会自动显示在界面上
  - 显示标注后的图片
  - 更新结果表格
  - 更新 OK/NG 状态
  - 更新缺陷数量统计

### 6. 停止监听
点击 **「⏹️ 停止监听」** 按钮即可停止监听。

## 技术细节

### 线程模型
- **实时监听模式**（watchdog）：
  - 使用系统文件系统事件通知
  - 文件变化立即响应（毫秒级）
  - 使用独立的后台线程处理事件
  - 所有 UI 更新通过 `QTimer.singleShot` 在主线程执行

- **扫描监听模式**（polling）：
  - 监听线程在后台独立运行
  - 使用 `threading.Event` 来控制线程停止
  - 所有 UI 更新通过 `QTimer.singleShot` 在主线程执行

### 文件处理逻辑
- **实时监听模式**：
  1. 监听文件系统事件
  2. 检测到文件创建/修改事件
  3. 短暂等待文件写入完成（0.3 秒）
  4. 在主线程调用检测插件
  5. 更新界面显示检测结果
  6. 记录已处理文件，避免重复处理

- **扫描监听模式**：
  1. 定期扫描文件夹（默认每 2 秒）
  2. 识别新文件（通过 `processed_files` 集合去重
  3. 等待文件写入完成（0.5 秒）
  4. 在主线程调用检测插件
  5. 更新界面显示检测结果
  6. 记录已处理文件，避免重复处理

### 窗口关闭处理
- 窗口关闭时自动停止监听线程
- 防止线程泄露

## 测试建议

1. **准备测试图片**
   - 使用 `generate_test_images.py` 生成测试图片
   - 或者使用自己的图片

2. **分阶段测试**
   - 先测试选择文件夹功能
   - 再测试启动/停止监听
   - 最后测试新文件自动检测

3. **模拟新文件**
   - 在监听的文件夹中添加新图片
   - 观察是否自动检测
   - 检查结果显示是否正确

## 注意事项

1. **初始文件处理**
   - 选择文件夹时会扫描现有图片
   - 现有图片会标记为「已处理」，不会重复检测
   - 只有新添加的图片会被自动检测

2. **文件写入延迟**
   - 检测到新文件后会等待 0.5 秒
   - 确保文件完全写入

3. **插件依赖**
   - 必须先选择插件才能启动监听
   - 监听过程中更换插件需要重启监听

4. **性能考虑**
   - 扫描间隔为 2 秒，可根据需要调整
   - 大文件夹可能会增加扫描时间
