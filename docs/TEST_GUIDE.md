# 界面测试指南

本文档介绍如何使用模拟插件和测试图片来测试界面显示效果。

---

## 📋 测试前准备

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 生成测试图片

```bash
python generate_test_images.py
```

这将在 `test_images/` 目录下生成15张测试超声图片：
- 5张正常图片（normal）
- 5张带缺陷图片（with_defects）
- 5张复杂缺陷图片（complex）

---

## 🚀 测试步骤

### 步骤1：运行软件

```bash
python bearing_defect_detection.py
```

### 步骤2：加载模拟插件

软件启动时会自动加载插件，确认右侧面板的"选择检测插件"下拉框中包含：
- ✅ **Mock Detector** - 模拟检测器

### 步骤3：选择测试图片

**方式1：选择单个图片**
1. 点击左侧"📷 图片选择"
2. 导航到 `test_images/` 目录
3. 选择任意一张图片

**方式2：选择整个文件夹（推荐）**
1. 点击左侧"📁 文件夹选择"
2. 导航到 `test_images/` 目录
3. 选择该文件夹
4. 系统会自动扫描并加载所有图片

### 步骤4：选择插件

在右侧面板的"选择检测插件"下拉框中，选择 **"Mock Detector"**

### 步骤5：开始测试

点击中央区域的 **"⚡ 立即开始"** 按钮

---

## 🎯 测试内容

### 1. **OK/NG状态显示**

测试不同状态下的显示效果：

| 场景 | 预期显示 | 触发方式 |
|------|---------|---------|
| 无缺陷 | 🟢 **OK**（绿色） | 选择 normal 类型图片 |
| 有缺陷 | 🔴 **NG**（红色） | 选择 with_defects 或 complex 类型 |
| 检测中 | 🟡 **检测中**（橙色） | 点击开始按钮后 |

### 2. **图片显示**

观察中间区域的图片显示：
- ✅ 是否显示检测图片
- ✅ 是否显示标注框（彩色矩形）
- ✅ 是否显示缺陷标签（类别 + 置信度）
- ✅ 是否显示序号

### 3. **缺陷数量**

观察顶部"缺陷数量"标签：
- 是否实时更新
- 数值是否正确

### 4. **结果表格**

观察底部表格：
- 是否显示检测结果
- 列是否完整（序号、路径、类别、置信度、坐标）

### 5. **统计信息**

观察顶部状态栏：
- ✅ 检测耗时
- ✅ 检测目标数

## 🔧 模拟插件配置

### 检测模式

Mock Detector 支持三种检测模式：

#### 模式1：随机模式（默认）
```python
# 70%概率返回缺陷，30%概率返回OK
plugin.detection_mode = 'random'
```

#### 模式2：总是OK
```python
# 总是返回无缺陷（OK）
plugin.detection_mode = 'always_ok'
```

#### 模式3：总是NG
```python
# 总是返回有缺陷（NG）
plugin.detection_mode = 'always_ng'
```

### 修改模式

如果需要修改检测模式，编辑 `plugins/detectors/mock_detector.py`：

```python
def __init__(self):
    super().__init__()
    # ...
    self.detection_mode = 'always_ng'  # 修改这里
```

## ⚠️ 常见问题

### 问题1：图片不显示

**原因**：缺少 OpenCV

**解决**：
```bash
pip install opencv-python
```

### 问题2：插件未加载

**检查**：
1. 确认 `plugins/detectors/mock_detector.py` 文件存在
2. 重启软件
3. 查看终端输出

### 问题3：图片加载慢

**原因**：图片数量太多

**解决**：减少 test_images 文件夹中的图片数量

## 🧹 清理测试数据

测试完成后，清理测试数据：

```bash
# 删除测试图片目录
rm -rf test_images/

# 删除 __pycache__
find plugins -name "__pycache__" -type d -exec rm -rf {} +
```
